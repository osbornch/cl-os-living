# os-living

> 知行合一。Model 是知，Reality 是行，系统的任务是让两者对齐。

**核心驱动：** 永远不必因为钱接受我不认同的条件。
**锚点：** 46岁，完全自主地活着。

---

## 怎么用这个系统

**每天开始：**
1. 打开 `orchestrator/snapshot.md` — 30秒重建上下文
2. 看 `orchestrator/DAG.md` — 根据能量和时间决定做什么
3. 做完一件事 → 一句话记到对应的 `Reality/` 文件夹

**每周日：**
- 打开当前 Quarter 文件（如 `orchestrator/Q3-2026.md`）
- 回答3个问题，更新 snapshot

**每季度末：**
- 复盘当前 Quarter，复制 `QUARTER_TEMPLATE.md` 开启下一个

**每年12月：**
- 运行 `YEAR_TEMPLATE.md`，校准方向

## 系统结构

```
os-living/
├── Goal.md                        ← 为什么做这一切（先读这里）
│
├── orchestrator/                  ← 运行层（每天在这里）
│   ├── snapshot.md                ← 当前系统状态
│   ├── DAG.md                     ← 每日决策树
│   ├── ROADMAP.md                 ← 10年4阶段路径
│   ├── Q3-2026.md                 ← 当前 Quarter
│   ├── QUARTER_TEMPLATE.md        ← 季度模板
│   └── YEAR_TEMPLATE.md           ← 年度校准模板
│
├── Model/                         ← 知（规则、框架、学习内容）
│   ├── core/                      ← 系统核心设计
│   ├── external_system/           ← 各域学习内容
│   │   ├── investment/
│   │   ├── design/
│   │   ├── history/
│   │   ├── literature/
│   │   └── psychology/
│   ├── inner_system/              ← 身体 + 精神
│   │   └── health/
│   └── explore/                   ← 20% 探索轮换（当前：biology）
│
├── Reality/                       ← 行（执行记录，每次做完写一句话）
│   ├── Create/                    ← 高能量生产输出
│   ├── Search/                    ← 研究、阅读、发现
│   ├── Align/                     ← 计划、沟通、协作
│   └── Upgrade/                   ← 学习、练习、复盘
│
└── Wisdom/                        ← 从 Reality 蒸馏出的原则
```
