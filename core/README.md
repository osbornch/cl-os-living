# os-living / core

The design spec for a personal life operating system.

---

## Files

| File | Purpose |
| ---- | ------- |
| `Goal.md` | Three core principles — why this system exists |
| `Architecture.md` | Layered OS design: Infrastructure → Hardware → Resource → Mind |
| `Input_agents.md` | Five cognitive modes — how to match mental state to task type |
| `schedule.md` | Energy-tiered action framework — what to do at each energy level |
| `mind_exercise.md` | Knowledge portfolio — 50/30/20 allocation + 20% exploration protocol |
| `Output.md` | Production layer — output types and how each one compounds back |
| `Review.md` | Feedback loop — weekly/monthly review protocol and parameter change rules |

---

## Reading Order (if new to this system)

1. `Goal.md` — understand the philosophy
2. `Architecture.md` — understand the structure and its failure modes
3. `schedule.md` + `Input_agents.md` — understand how to operate day-to-day
4. `mind_exercise.md` — understand how to allocate learning energy
5. `Output.md` — understand what to produce and why
6. `Review.md` — understand how to maintain and correct the system
