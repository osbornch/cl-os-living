对。**做题只是验证的一种，不是唯一的方法。**

可以把「学习 → 成果 → 验证」理解成不同层级的反馈系统。

# 1. 做题（Solve）

适合：

* 数学
* 算法
* 语言
* 基础知识

验证：

> 我会不会？

特点：

* 反馈快
* 边界清晰
* 但容易停留在「会做题」

---

# 2. 做项目（Build）

适合：

* 编程
* AI
* 产品
* 设计
* 摄影
* 音乐

验证：

> 我能不能从 0 到 1 做出来？

例如：

学习 RAG

不是刷 100 道题，而是：

```
PDF → chunk → embedding → retrieval → answer
```

做一个完整 Demo。

项目会暴露：

* 真正不会的地方
* 模块如何组合
* 工程能力

---

# 3. 输出（Teach）

费曼学习法。

验证：

> 我能让别人懂吗？

方式：

* 写博客
* 做笔记
* 录视频
* 给 AI 讲
* 给同事分享

如果讲不清：

说明理解还停留在碎片。

---

# 4. 实战（Perform）

这是更高一级的验证。

例如：

Sales：

* 真去约客户

英语：

* 真和人交流

网球：

* 真比赛

跳舞：

* 真上台

系统设计：

* 真处理生产问题

验证：

> 压力下还能不能用出来？

---

# 5. 时间验证（Retention）

很多东西不是会，而是能不能留下。

例如：

一周后：

还能记住多少？

一个月后：

还能重新做出来吗？

三个月后：

还能不用查资料讲出来吗？

这验证：

> 是否进入长期记忆。

---

# 6. 迁移验证（Transfer）

这是高手和初学者最大的区别。

验证：

> 能不能在新问题上复用？

例如：

学会 Kubernetes

不是会部署 YAML。

而是：

能不能迁移到：

* Ray
* Agent
* Databricks
* 分布式系统

学会篮球

能不能迁移到：

* 足球
* 网球
* 舞蹈

学会 Sales

能不能迁移到：

* 面试
* 管理
* 育儿
* 谈判

迁移能力说明形成了抽象模型。

---

# 7. 创造（Create）

最高层。

验证：

> 能不能产生别人没有教过的东西？

例如：

不是学 LangGraph。

而是设计：

* 自己的 Agent Framework
* 自己的 Memory System
* 自己的 Prompt Pattern

不是学健身。

而是设计：

* 自己的训练系统。

---

# 一个完整的验证金字塔

```
        Create
          ↑
       Transfer
          ↑
       Perform
          ↑
        Teach
          ↑
        Build
          ↑
        Solve
```

对应的问题：

| 层级       | 验证问题     |
| -------- | -------- |
| Solve    | 我会吗？     |
| Build    | 我能做出来吗？  |
| Teach    | 我能解释吗？   |
| Perform  | 压力下能用吗？  |
| Transfer | 新场景能用吗？  |
| Create   | 能产生新东西吗？ |

所以学习并不是：

```
学习 → 做题 → 结束
```

而更像：

```
学习
 ↓
做题（局部验证）
 ↓
做项目（系统验证）
 ↓
输出（结构验证）
 ↓
实战（压力验证）
 ↓
迁移（抽象验证）
 ↓
创造（创新验证）
```

对于编程、AI、产品、投资、运动这些复杂能力，**做项目往往比做题更接近真实能力，但真正强大的验证方式，是「在真实世界创造价值」**。
